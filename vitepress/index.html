创建、构建与部署 #

创建 #

mkdir vitepress-starter && cd vitepress-starter

yarn init

yarn add --dev vitepress

mkdir docs && echo '# Hello VitePress' > docs/index.md

在 package.json 中新增脚本

"scripts": {
    "docs:dev": "vitepress dev docs",
    "docs:build": "vitepress build docs",
    "docs:serve": "vitepress serve docs"
}

本地启动服务

yarn docs:dev

构建 #

执行 yarn docs:build 命令构建 docs 文档

yarn docs:build

默认情况下，构建输出在 .vitepress/dist 目录

测试 #

构建文档之后，你可以通过运行 yarn docs:serve 命令在本地测试

yarn docs:serve

serve 命令将启动本地静态 Web 服务器 http://localhost:5000，服务器内容从 .vitepress/dist 中获取。可以用来检查生产版本在本地环境中是否正常。

可以通过 --port 参数来配置服务器端口

"scripts": {
    "docs:serve": "vitepress serve docs --port 8080"
}

现在执行 docs:serve 将在 http://localhost:8080 启动服务器。

部署 #

以部署到 GitHub Pages 为例

  1. docs/.vitepress/config.js 中设置正确的 base 参数

    如果部署到 https://<USERNAME>.github.io/，你可以省略 base，因为它默认为 '/'

    如果部署到 https://<USERNAME>.github.io/<REPO>/，则设置 base 到 '/<REPO>/'

  2. 在项目根目下创建 deploy.sh (突出显示的行适当取消注释)，然后运行进行部署：













 






 


 



#!/usr/bin/env sh

# abort on errors
set -e

# build
npm run docs:build

# navigate into the build output directory
cd docs/.vitepress/dist

# if you are deploying to a custom domain
# echo 'www.example.com' > CNAME

git init
git add -A
git commit -m 'deploy'

# if you are deploying to https://<USERNAME>.github.io
# git push -f git@github.com:<USERNAME>/<USERNAME>.github.io.git master

# if you are deploying to https://<USERNAME>.github.io/<REPO>
# git push -f git@github.com:<USERNAME>/<REPO>.git master:gh-pages

cd -

TIP

您还可以在 CI 设置中运行上述脚本，以在每次推送时启用自动部署
