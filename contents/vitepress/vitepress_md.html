Markdown 扩展 #

资源处理 #

所有 Markdown 文件都编译成 Vue 组件并由 Vite 处理。您应该使用相对 URL 引用资源，比如：

![An image](./image.png)

您可以在 Markdown 文件、主题中的 *.vue 组件、styles和 .css 中使用绝对公共路径（基于项目根目录）或相对路径（基于文件系统）引用静态资源。

所有引用的资源（包括使用绝对路径的资源）都将复制到生产版本中具有哈希文件名的 dist 文件夹。从未引用的资源不会被复制。与 vue-cli 类似，小于 4kb 的图像资源将以 base64 内联。

所有静态路径引用（包括绝对路径）都应基于工作目录结构。

公共文件

有时，您可能需要提供在任何 Markdown 或主题组件中未直接引用的静态资源（例如，favicons 和 PWA icons）。项目根目录(docs/)下的 public 文件夹可以用来放置那些在源文件总从来不会被引用的静态资源（比如，robots.txt），或者必须保留原文件名（不HASH）的文件。

放入 public 文件夹下的资源将按原样复制到 dist 目录的根目录中。

请注意，您应该引用使用 root absolute path 来引用 public 文件夹下的资源 - 例如，public/icon.png 在源码总应该被引用为 /icon.png。

Base路径

如果你的网站被部署到了一个非根目录的路径，你需要在 .vitepress/config.js 总设置 base 选项。比如，如果你打算将你的网站部署到 https://foo.github.io/bar/，那么 base 应该设置为 '/bar/'（base 应该以反斜杠开始和结束）。

所有静态资源路径会根据不同的 base 选项自动进行调整。例如，如果您在 Markdown 中使用绝对路径引用了 public 下的资源：

![An image](/image-inside-public.png)

当您更改 base 配置值时，您不需要更新它。 但是，如果您正在制作一个【需要动态链接到资源】的主题组件，例如，一个图像，其 src 基于主题配置值：

<img :src="theme.logoPath" />

在这种情况下，建议使用 VitePress 提供的 withBase 来包裹路径：

<script setup>
import { withBase, useData } from 'vitepress'
const { theme } = useData()
</script>

<template>
    <img :src="withBase(theme.logoPath)" />
</template>

链接 #

内部链接

内部链接（Internal links）转换为路由器链接（router link），用于 SPA 导航。此外，每个子目录中包含的每个 index.md 都会自动转换为 index.html，并带有相应的URL /。

例如，给定以下目录结构：

.
├─ index.md
├─ foo
│  ├─ index.md
│  ├─ one.md
│  └─ two.md
└─ bar
   ├─ index.md
   ├─ three.md
   └─ four.md

假设你在 foo/one.md:

[Home](/)                     <!-- 回到 根目录 下的 index.md -->
[foo](/foo/)                  <!-- 回到 foo 下的 index.md -->
[foo heading](./#heading)     <!-- 定位到 foo/index.md 中的 heading 标题 -->
[bar-three](../bar/three)     <!-- 可以省略扩展名 -->
[bar-three](../bar/three.md)  <!-- 指定扩展名 .md -->
[bar-four](../bar/four.html)  <!-- 指定扩展名 .html -->

文件后缀

默认情况下，页面和内部链接生成的时候使用 .html 后缀。

外部链接

外部链接自动添加 target="_blank" rel="noopener noreferrer":

  • VitePress on GitHub

代码语法高亮 #

VitePress 使用 Prism 进行 Markdown 代码块语法高亮。Prism 支持多种编程语言。只需要在代码块的开头附加一个语言别名就行了。

输入

```js
export default {
    name: 'MyComponent',
    // ...
}
```

输出

export default {
    name: 'MyComponent'
    // ...
}

输入

```html
<ul>
    <li v-for="todo in todos" :key="todo.id">
        {{ todo.text }}
    </li>
</ul>
```

输出

<ul>
    <li v-for="todo in todos" :key="todo.id">
        {{ todo.text }}
    </li>
</ul>

代码块行高亮 #

输入

```js{4}
export default {
    data () {
        return {
            msg: 'Highlighted!'
        }
    }
}
```

输出




 




export default {
    data () {
        return {
            msg: 'Highlighted!'
        }
    }
}

除了单行之外，还可以指定多行，范围，或者混用

  • 行范围高亮: {5-8}, {3-10}, {10-17}
  • 多个单行高亮: {4,7,9}
  • 混用: {4,7-13,16,23-27,40}

输入

```js{1,4,6-7}
export default { // Highlighted
    data () {
        return {
            msg: `Highlighted!
            This line isn't highlighted,
            but this and the next 2 are.`,
            motd: 'VitePress is awesome',
            lorem: 'ipsum',
        }
    }
}
```

输出

行号 #

可以配置启用代码块的行号显示

module.exports = {
    markdown: {
        lineNumbers: true
    }
}

TIP

对于 ```` 中嵌套 ``` 行号显示不太完美，会有错位。

示例

导入代码片段 #

可以通过以下语法从现有文件中导入代码片段

<<< @/filepath

导入的代码片段也支持行高亮

<<< @/filepath{highlightLines}

输入

<<< @/snippets/snippet.js{2}

代码文件

export default function () {
    // ..
}

输出


 


export default function () {
    // ..
}

TIP

@ 的值对应于根目录。 默认情况下它是 VitePress 项目根目录(docs)，除非配置了 srcDir。

也可以使用 VS Code region 来只包含代码文件的一部分。 可以在文件路径后的 # 跟一个自定义范围(默认为 snippet):

输入

<<< @/snippets/snippet-with-region.js{1}

代码文件

// #region snippet
function foo() {
    // ..
}
// #endregion snippet

export default foo

输出

 


function foo() {
    // ..
}

进阶配置 #

VitePress 使用 markdown-it 来渲染 Markdown。上面的很多扩展都是通过自定义插件实现的。 你可以通过 .vitepress/config.js 中的 markdown 选项进一步自定义 markdown-it。

const anchor = require('markdown-it-anchor')

module.exports = {
    markdown: {
        // options for markdown-it-anchor
        // https://github.com/valeriangalliat/markdown-it-anchor#permalinks
        anchor: {
            permalink: anchor.permalink.headerLink()
        },

        // options for markdown-it-table-of-contents
        toc: { includeLevel: [1, 2] },

        config: (md) => {
            // use more markdown-it plugins!
            md.use(require('markdown-it-xxx'))
        }
    }
}
