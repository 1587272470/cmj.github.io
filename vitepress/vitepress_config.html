配置 #

首先在您的 docs 目录中创建一个 .vitepress 目录。 这是所有 VitePress 特定文件将被放置的地方。你的项目结构大概是这样的：

.
├─ docs
│  ├─ .vitepress
│  │  └─ config.js
│  └─ index.md
└─ package.json

配置文件为 .vitepress/config.js，应该导出一个 JavaScript 对象：

module.exports = {
    title: "CMJ的笔记",
    description: 'CMJ的笔记',
}

base

  • Type: string
  • Default: /

网站被部署到的基本 URL。base 会被自动添加到其他选项中以 / 开头的所有 URL，因此只需要指定一次。

module.exports = {
    base: '/base/'
}

lang

  • Type: string
  • Default: en-US

网站的 lang 属性，和 HTML 中的 <html lang="en-US"> 标签类似。

module.exports = {
    lang: 'en-US'
}

title

  • Type: string
  • Default: VitePress

网站的标题。 所有页面标题的后缀。并显示在导航栏中。

module.exports = {
    title: 'VitePress'
}

description

  • Type: string
  • Default: A VitePress site

网站说明。和 HTML 的 <meta> 标签类似。

module.exports = {
    description: 'A VitePress site'
}

主页布局 #

VitePress 提供了主页布局。在根目录下的 index.md YAML frontmatter 中指定 home: true。

示例：

---
home: true
heroImage: /logo.png
heroAlt: Logo image
heroText: Hero Title
tagline: Hero subtitle
actionText: Get Started
actionLink: /guide/
features:
  - title: Simplicity First
    details: Minimal setup with markdown-centered project structure helps you focus on writing.
  - title: Vue-Powered
    details: Enjoy the dev experience of Vue + webpack, use Vue components in markdown, and develop custom themes with Vue.
  - title: Performant
    details: VitePress generates pre-rendered static HTML for each page, and runs as an SPA once a page is loaded.
footer: MIT Licensed | Copyright © 2019-present Evan You
---
