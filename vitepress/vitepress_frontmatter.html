Frontmatter #

所有包含 YAML frontmatter 块的 Markdown 文件将由 gray-matter 处理。frontmatter 必须位于 Markdown 文件的顶部，且必须采用有效的 YAML 格式（---）

---
title: Docs with VitePress
editLink: true
---

在三横线之间，你可以设置预定义属性，甚至自定义变量。这些变量可以通过特殊的 $frontmatter 变量使用。

---
title: Docs with VitePress
editLink: true
---

# {{ $frontmatter.title }}

Guide content

JSON 格式 #

VitePress 同样支持 JSON 格式的 frontmatter 语法，以花括号开头和结尾：

---
{
    "title": "Blogging Like a Hacker",
    "editLink": true
}
---

预定义属性 #

title

  • Type: string
  • Default: h1_title || siteData.title

head

  • Type: array
  • Default: undefined

指定要注入的额外头部标签

---
head:
  - - meta
    - name: description
      content: hello
  - - meta
    - name: keywords
      content: super duper SEO
---

navbar

  • Type: boolean
  • Default: undefined

你可以使用 navbar: false 在一些界面禁用导航条。

sidebar

  • Type: boolean|'auto'
  • Default: undefined

显示侧边栏 sidebar: auto，禁用侧边栏 sidebar: false

editLink

  • Type: boolean
  • Default: undefined

页面是否应包含编辑链接。
