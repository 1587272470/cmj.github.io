Markdown 中使用 Vue #

在 VitePress 中，每个 Markdown 文件都被编译成 HTML，然后作为 Vue 单文件组件进行处理。 这意味着您可以在 Markdown 中使用任何 Vue 功能，包括动态模板、使用 Vue 组件或通过添加 <script> 标签使用页面内 Vue 组件。

VitePress 利用 Vue 3 的编译器来自动检测和优化 Markdown 的纯静态部分。 静态内容被优化为单个占位符节点，并从页面的 JavaScript 负载中消除。 简而言之，您只需关注页面内的动态部分。

示例 #

插值 #

每个 Markdown 文件首先被编译成 HTML，然后作为 Vue 组件传递到 Vite 进程管线。 这意味着您可以在文本中使用 Vue 风格的插值：

输入

{{ 1 + 1 }}

输出

2

指令 #

指令同样生效

输入

<span v-for="i in 3">{{ i }} </span>

输出

1 2 3

访问站点和页面数据 #

你可以在 <script> 块中使用 useData 将数据公开给页面

输入

<script setup>
    import { useData } from 'vitepress'
    const { page } = useData()
</script>

<pre>{{ page }}</pre>

输出

{
    "path": "/using-vue.html",
    "title": "Using Vue in Markdown",
    "frontmatter": {}
}

Escaping #

默认情况下，受保护的代码块会自动用 v-pre 包装。 要在内联代码片段或纯文本中显示原始语法或特定于 Vue 的语法，您需要使用 v-pre 自定义容器包装一个段落：

输入

::: v-pre
`{{ This will be displayed as-is }}`
:::

输出

{{ This will be displayed as-is }}

使用组件 #

当您需要更大的灵活性时，VitePress 允许您使用自己的 Vue 组件扩展您的工具箱。

导入组件 #

如果您的组件仅在少数几个地方使用，推荐将组件导入到使用它的文件中。

<script setup>
    import CustomComponent from '../components/CustomComponent.vue'
</script>

# Docs

This is a .md using a custom component

<CustomComponent />

## More docs

...

在主题中注册全局组件 #

如果组件将在文档中的多个页面上使用，它们可以在主题中全局注册（或作为扩展默认 VitePress 主题的一部分）。 查看 主题 了解更多信息。

在 .vitepress/theme/index.js 中，enhanceApp 函数接收 Vue 的 app 实例，因此你可以 注册组件，就像您在常规 Vue 应用程序中所做的那样。

import DefaultTheme from 'vitepress/theme'

export default {
    ...DefaultTheme,
    enhanceApp({ app }) {
        app.component('VueClickAwayExample', VueClickAwayExample)
    }
}

注册之后就可以在所有的 markdown 文件中使用组件。

# Vue Click Away

<VueClickAwayExample />

重要

确保自定义组件的名称包含 hyphen 或采用 PascalCase。 否则，它将被视为内联元素并包裹在 <p> 标签内，这将导致 hydration 不匹配，因为 <p> 不允许将块元素放置在其中。

在标题中使用组件 ⚡ #

您可以在标题（headers）中使用 Vue 组件，但请注意以下语法之间的区别：

MarkdownOutput HTMLParsed Header
 # text <Tag/>
<h1>text <Tag/></h1>text
 # text `<Tag/>`
<h1>text <code>&lt;Tag/&gt;</code></h1>text <Tag/>

<code> 包裹的 HTML 将按原样显示； 没有被包装的 HTML 才会被 Vue 解析。

TIP

输出 HTML 由 markdown-it 完成，而解析标题（headers）由 VitePress 处理（并用于侧边栏和文档标题）。

使用 CSS 预处理器 #

VitePress 为 CSS 预处理器文件(.scss、.sass、.less、 .styl 和 .stylus)提供了 built-in support。 不需要为它们安装 Vite 特定的插件，但必须安装相应的预处理器本身：

# .scss and .sass
npm install -D sass

# .less
npm install -D less

# .styl and .stylus
npm install -D stylus

然后你可以在 Markdown 和主题组件中使用以下内容：

<style lang="sass">
.title
    font-size: 20px
</style>

内置组件 #

VitePress 提供了内置的全局 Vue 组件，如 Content ClientOnly 和 OutboundLink，您可以在 Markdown 或自定义主题配置中使用这些组件。

Content

Content 组件显示 markdown 内容，当创建自定义主题时非常有用。

<template>
    <h1>Custom Layout!</h1>
    <Content />
</template>

ClientOnly

ClientOnly 组件仅在客户端呈现。

<ClientOnly>
    <NonSSRFriendlyComponent />
</ClientOnly>

OutboundLink

OutboundLink 用于表示外部链接。 在 VitePress 中，每一个外部链接前面都会跟一个 OutboundLink 组件。

浏览器 API 访问限制 #

由于 VitePress 应用程序在生成静态构建时在 Node.js 中进行服务器渲染，因此任何 Vue 使用都必须符合 通用代码要求。 简而言之，请确保仅在 beforeMount 或 mounted 钩子中访问 Browser / DOM API。

如果您正在使用或演示对 SSR 不友好的组件（例如，包含自定义指令），您可以将它们包装在内置的 <ClientOnly> 组件中：

<ClientOnly>
    <NonSSRFriendlyComponent/>
</ClientOnly>

请注意，这不会修复在导入时访问浏览器 API 的组件或库。 要在导入时使用假定浏览器环境的代码，您需要在适当的生命周期钩子中动态导入它们：

<script>
    export default {
        mounted() {
            import('./lib-that-access-window-on-import').then((module) => {
                // use code
            })
        }
    }
</script>

如果你的模块 export default 是一个 Vue 组件，你可以动态注册它：

<template>
    <component v-if="dynamicComponent" :is="dynamicComponent"></component>
</template>

<script>
    export default {
        data() {
            return {
                dynamicComponent: null
            }
        },

        mounted() {
            import('./lib-that-access-window-on-import').then((module) => {
                this.dynamicComponent = module.default
            })
        }
    }
</script>
